# Luiza Estate

> **North Star:** построить один из самых доверенных персональных брендов в сфере недвижимости Москвы и создать вокруг экспертизы Луизы высокорентабельный бизнес, усиленный технологиями.

Luiza Estate — это не только сайт-визитка риелтора.

Цель проекта — последовательно развивать персональный бренд Луизы Алиниседовой, улучшать привлечение и конверсию клиентов, систематизировать знания о бизнесе и создавать цифровую инфраструктуру для дальнейшего роста.

Сайт является одним из ключевых инструментов продукта, но не единственным. Репозиторий хранит как код приложения, так и продуктовую документацию: стратегию, исследования, SEO-аудиты, дорожную карту и принятые решения.

## Видение

Мы строим:

- сильный и узнаваемый персональный бренд;
- понятный и конверсионный сайт;
- устойчивую систему привлечения качественных обращений;
- базу знаний о бизнесе, клиентах и рынке;
- измеримую воронку от первого контакта до закрытой сделки;
- основу для возможного роста за пределы бизнеса одного специалиста.

## Текущее состояние

Сейчас проект представляет собой статически генерируемый сайт на Astro. Основной контент
попадает непосредственно в HTML, а клиентский JavaScript не требуется для чтения страницы,
навигации и контактных действий.

Ближайший этап — укрепить SEO-фундамент, устранить технические ограничения текущей реализации и перезапустить главную страницу как полноценную посадочную страницу специалиста.

## Технологии

- Node.js 22
- Astro 6
- TypeScript
- CSS
- Google Analytics

Текущая архитектура будет пересматриваться с учётом SEO, скорости загрузки и необходимости серверного или статического рендеринга публичных страниц.

## Разработка

- [`AGENTS.md`](AGENTS.md) — обязательные правила для AI-агентов;
- [`CONTRIBUTING.md`](CONTRIBUTING.md) — правила веток и Pull Request;
- [`docs/development/README.md`](docs/development/README.md) — документация по локальной разработке, workflow и release.

Основная единица работы — GitHub Issue. Обычные задачи выполняются в ветках `agent/*` и направляются в `develop`. Ветка `main` соответствует production и обновляется через release PR `develop → main`.

## Текущие приоритеты

1. Завершить базовое исследование бизнеса и цифрового присутствия.
2. Выполнить SEO Foundation Sprint.
3. Развивать статическую Astro-архитектуру публичного сайта и её SEO-проверки.
4. Перезапустить главную страницу.
5. Создать отдельные страницы ключевых услуг.
6. Настроить аналитику обращений и конверсий.
7. Развивать контент, кейсы, отзывы и органический трафик.

## Принципы проекта

Мы оптимизируем решения под:

- доверие;
- ясность;
- измеримый бизнес-результат;
- высокую рентабельность;
- удобство клиента;
- поддерживаемость;
- долгосрочный рост.

Неизвестные факты не выдаются за подтверждённые. Гипотезы, исследования и важные решения фиксируются в документации.

## Рабочий процесс

Для существенных изменений используется следующий процесс:

1. Создание или уточнение GitHub Issue.
2. Фиксация продуктового или архитектурного решения, если оно требуется.
3. Работа в отдельной ветке `agent/*` от `develop`.
4. Draft Pull Request в `develop`.
5. CI, AI-review и review мейнтейнера.
6. Merge в `develop`.
7. Выпуск через PR `develop → main`.

## Долгосрочная цель

Построить высокорентабельный, технологичный и устойчивый бизнес в сфере недвижимости, который опирается на экспертность Луизы, персональное доверие, системный маркетинг и качественный клиентский сервис.
