# Дорожная карта продукта

## Принципы приоритизации

Приоритет получают изменения, которые:

1. повышают доверие;
2. делают услуги понятнее;
3. увеличивают число качественных обращений;
4. дают измеримый результат;
5. создают основу для дальнейшего продвижения.

## Этап 0 — исследование и подготовка

Статус: следующий этап.

Задачи:

- провести интервью с Луизой;
- уточнить приоритетные услуги и нежелательные типы обращений;
- собрать актуальные контакты;
- подтвердить опыт, специализацию и географию;
- собрать фотографии;
- подготовить 3–5 реальных кейсов;
- собрать отзывы и разрешения на публикацию;
- определить условия первой консультации;
- проверить текущую аналитику, домен и способ публикации сайта.

Результат: подтверждённый контентный бриф и список материалов для главной страницы.

## Этап 1 — перезапуск главной страницы

Цель: превратить визитку в полноценную посадочную страницу специалиста.

Задачи:

- сформулировать позиционирование;
- переработать первый экран;
- представить услуги через клиентские ситуации;
- добавить блок доверия;
- описать процесс работы;
- добавить первые кейсы и отзывы;
- переработать контактный сценарий;
- улучшить мобильную версию;
- настроить события аналитики;
- обновить метаданные и превью для социальных сетей;
- добавить обязательные страницы по персональным данным.

Критерий готовности: посетитель понимает специализацию, видит доказательства опыта и может обратиться удобным способом.

## Этап 2 — страницы приоритетных услуг

Цель: создать релевантные точки входа для поиска, рекламы и рекомендаций.

Задачи:

- выбрать 2–3 наиболее важных направления;
- подготовить отдельные страницы;
- добавить для каждой услуги процесс, риски, ответы на вопросы и CTA;
- настроить внутренние ссылки;
- добавить SEO-метаданные и структурированные данные;
- измерять обращения отдельно по каждой странице.

Критерий готовности: для приоритетной клиентской задачи существует отдельная убедительная страница.

## Этап 3 — кейсы и контент доверия

Цель: регулярно подтверждать экспертизу практикой.

Задачи:

- создать единый шаблон кейса;
- публиковать новые кейсы;
- оформить отзывы;
- подготовить ответы на частые вопросы;
- при необходимости запустить экспертные статьи;
- связать публикации с Telegram, VK или другими используемыми каналами.

Критерий готовности: сайт содержит достаточно реальных материалов, чтобы клиент мог оценить подход до консультации.

## Этап 4 — управление обращениями

Цель: не терять лиды и понимать эффективность каналов.

Возможные задачи:

- форма с квалифицирующими вопросами;
- уведомления о новых обращениях;
- фиксация источника и UTM-меток;
- простая таблица или CRM для статусов;
- шаблоны первого ответа;
- отчёт по обращениям и конверсии.

Реализация этого этапа зависит от реального объёма лидов. Не следует внедрять сложную CRM заранее.

## Этап 5 — привлечение трафика

Возможные направления:

- локальное SEO;
- Яндекс Бизнес и карты;
- контекстная реклама по конкретным услугам;
- ретаргетинг;
- публикации в социальных сетях;
- партнёрские рекомендации;
- полезные материалы для существующих клиентов и их знакомых.

Продвижение запускается после того, как страницы способны конвертировать трафик в обращения и аналитика работает корректно.
